# دليل تسليم مشروع HandyLand (Handover Guide)

هذا المستند مخصص للمبرمج الذي سيتولى تطوير أو صيانة مشروع **HandyLand**. يشرح هذا الدليل هيكلية المشروع، التقنيات المستخدمة، ومنطق العمل الأساسي من الألف إلى الياء.

---

## 1. نظرة عامة على المشروع
**HandyLand** هي منصة متكاملة للتجارة الإلكترونية والخدمات (مخصصة للسوق الألماني بشكل أساسي). تتيح للمستخدمين شراء الأجهزة، حجز خدمات التصلاح، وتقييم أجهزتهم القديمة لبيعها، مع لوحة تحكم إدارية كاملة.

---

## 2. مكدس التقنيات (Tech Stack)

### **الخلفية (Backend)**
*   **اللغة والبيئة:** Node.js مع Express.js.
*   **قاعدة البيانات:** MongoDB باستخدام Mongoose (Object Modeling).
*   **الأمان:**
    *   `jsonwebtoken` (JWT) للمصادقة.
    *   `bcryptjs` لتشفير كلمات المرور.
    *   `helmet` و `express-rate-limit` للحماية من الهجمات.
    *   `cors` و `xss-clean` لتأمين البيانات.
*   **الخدمات الخارجية:**
    *   `Stripe` و `PayPal` للمدفوعات.
    *   `Cloudinary` لتخزين وإدارة الصور.
    *   `Nodemailer` لإرسال البريد الإلكتروني.
    *   `Socket.io` للدردشة الفورية.

### **الواجهة الأمامية (Frontend - User & Admin)**
*   **إطار العمل:** React.js (الإصدار 18/19) مع **TypeScript**.
*   **أداة البناء:** Vite (لسرعة التطوير والأداء).
*   **التنسيق (Styling):** Tailwind CSS مع تصميم "Glassmorphism" وتفاعلات عصرية.
*   **الحركة:** `framer-motion` للحصول على تجربة مستخدم سلسة (Premium UX).
*   **إدارة الحالة:**
    *   `React Context API` للإعدادات والمصادقة.
    *   `TanStack Query` (React Query) لجلب البيانات من الـ API وإدارة الكاش.
*   **النماذج:** `react-hook-form` مع `zod` للتحقق من البيانات (Validation).

## 3. هيكلية الملفات (Directory Structure)

```text
handyland/
├── backend/                  # خادم API (Node.js)
│   ├── admin/                # تطبيق لوحة التحكم (React App منفصل)
│   ├── controllers/          # منطق العمل الخاص بكل مسار (Route Logic)
│   ├── models/               # تعريف جداول قاعدة البيانات (Schemas)
│   ├── routes/               # تعريف نهايات الـ API (Endpoints)
│   ├── middleware/           # برمجيات وسيطة (Auth, Error Handling, Uploads)
│   ├── utils/                # أدوات مساعدة (إرسال إيميلات، معالجة صور)
│   └── server.js             # نقطة الدخول الرئيسية للخادم
│
├── front-end/                # تطبيق الموقع الموجه للمستخدم (React App)
│   ├── src/
│   │   ├── components/       # المكونات القابلة لإعادة الاستخدام
│   │   ├── context/          # إدارة الحالة العالمية (Settings, Auth)
│   │   ├── pages/            # الصفحات الرئيسية للموقع
│   │   └── hooks/            # الدوال المخصصة (Custom Hooks)
```

## 4. المفاهيم الأساسية في المشروع

### **أولاً: نظام الإعدادات الديناميكي (Settings Architecture)**
هذا أهم جزء في المشروع. الموقع لا يعتمد على نصوص ثابتة (Hardcoded)، بل يتم جلب كل شيء (الشعار، الألوان، نصوص الـ SEO، روابط التواصل) من قاعدة البيانات.
*   المسؤول يغير الإعدادات من `SettingsManager` في لوحة التحكم.
*   يتم تخزينها في الـ Backend في موديل `Settings`.
*   يتم استهلاكها في الـ Frontend عبر `SettingsContext` وتخزينها في `localStorage` لسرعة التحميل.

### **ثانياً: دورة حياة الطلبات (Order Lifecycle)**
الطلبات تمر بمراحل (Pending, Processing, Shipped, Completed, Cancelled). يتم تحديث الحالة من لوحة التحكم، ويستلم المستخدم إشعارات فورية عبر البريد الإلكتروني.

### **ثالثاً: تعدد اللغات (Localization)**
المشروع مجهز بالكامل لدعم اللغات (خاصة الألمانية) باستخدام `i18next`. الترجمات موجودة في ملفات JSON ويتم تحميلها ديناميكياً.

## 5. كيفية التشغيل (Local Setup)

1.  **قاعدة البيانات:** تأكد من تشغيل MongoDB محلياً أو استخدام رابط Atlas.
2.  **الخلفية:**
    ```bash
    cd backend
    npm install
    npm run dev # يعمل على المنفذ 5000
    ```
3.  **لوحة التحكم:**
    ```bash
    cd backend/admin
    npm install
    npm run dev # يعمل على المنفذ 5173
    ```
4.  **الموقع الرئيسي:**
    ```bash
    cd front-end
    npm install
    npm run dev # يعمل على المنفذ 3000
    ```

## 6. ملاحظات هامة للمطور الجديد
*   **الأمان:** تم تفعيل `Strict Mode` في TypeScript، يرجى عدم تجاهل الأخطاء البرمجية.
*   **الصور:** يتم رفع الصور إلى Cloudinary، تأكد من وجود المفاتيح الصحيحة في ملف `.env`.
*   **التنسيق:** نستخدم Tailwind بشكل مكثف، يرجى مراجعة ملف `tailwind.config.js` لمعرفة المتغيرات الخاصة بالألوان.
